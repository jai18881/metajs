Application architecture

Image that we have project management application which manages projects and it's tasks.

How the UI should look like?

We need navigation and dynamic content where individal screens will be placed.

Here comes the Activity component which defines our entire app. Then navigation is Fragment and content is another Activity.

Project list

Now lets design project list.

We want to display all projects from database. Here comes Data provider which provides data from server and updates Activity.

Our list view consists of univeral list fragment and we specify fragment for each row so we can control list design.

When user clicks on a project then content activity will be switched to project detail with tasks.

Project detail

Now lets design project detail.

We want to display project info and it's tasks. We use another Data provider for manipulate with tasks.

Our detail consists of simple project info as HTML and univeral list with task row fragment.

Task row fragment can provide task maniuplation.

How data providers interacts?

Init

Data provider loads data from server and notifies listeners (fragments or activities) about data.

Action

User makes an action and then Activity or Fragment calls provider method.

Update

Provider takes and action processes request (calls API for example) and notifies listeners about changes so they can update views.

Summary

Activity defines specific application activity like project list or project detail.

Fragment defines reusable UI component like toolbar or project row. Lately we can use task row to show last task in project list.

Data provider provides data access and manipulation.

View defines reusable HTML template for each activity and fragment.

DOM template is library which adds data-binding and view-logic to our views.

Channels are implementation of publish / subscribe pattern.
